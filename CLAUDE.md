## Project Overview

**ft2026** is a React component library and learning documentation platform built with [Dumi v2](https://d.umijs.org/) and [Father](https://github.com/umijs/father). It publishes both an npm package (`ft2026`) and a GitHub Pages documentation site.

## Commands

### Development

```bash
npm run dev          # Start Dumi docs dev server at http://localhost:8000
npm start            # Alias for dev
npm run build:watch  # Watch mode for library source
```

### Building

```bash
npm run build        # Build library source to dist/ (ESM via Father)
npm run docs:build   # Build static documentation site
npm run docs:preview # Preview production docs build locally
```

### Code Quality

```bash
npm run lint         # ESLint with import sorting
npm run lint:fix     # ESLint with auto-fix
npm run format       # Prettier formatting (all files)
npm run format:check # Prettier check (no changes)
npm run typecheck    # TypeScript check (no emit)
```

### Publishing

```bash
npm run bump:beta    # Bump prerelease version
npm run publish:beta # Publish beta to npm
```

## Architecture

### Dual Output

The repo serves two purposes simultaneously:
1. **npm library** — source in `src/`, built to `dist/` via Father (ESM only)
2. **Documentation site** — markdown + live demos in `docs/`, built via Dumi and deployed to GitHub Pages

### Library Exports (`src/`)

Three named entry points defined in `.fatherrc.ts` and `package.json#exports`:
- `.` → `src/index.ts` — main re-exports
- `./app/settings` → `src/app/settings/index.ts` — `useSettings` hook (localStorage management)
- `./mocks/api-mocks` → `src/mocks/api-mocks.ts` — MSW setup helpers

Key modules:
- **`src/app/settings/`** — `useSettings` hook for typed localStorage settings
- **`src/infrastructure/net/`** — Axios instances (`authApi`, `coreApi`) + mock JWT utilities (`jose`)
- **`src/mocks/`** — MSW worker setup with base path handling + file verification helpers

### Documentation (`docs/`)

- `docs/threads/` — Learning articles, each typically pairing a markdown explanation with a live React playground embedded in the `.md` file
- `docs/public/mockServiceWorker.js` — MSW service worker (must stay here for browser mocking in demos)
- Base path is configured for GitHub Pages subdirectory (`/frontend-threads-2026/`)

### TypeScript Path Aliases

- `@/*` → `src/*`
- `@@/*` → `.dumi/*`
- `ft2026` → `src/index.ts` (allows docs to import the library as if it were installed)

### CI/CD

GitHub Actions (`.github/workflows/cicd.yml`) runs on pushes/PRs to `main`:
lint → typecheck → build src → build docs → deploy to GitHub Pages (Node 24).

## Key Dependencies

| Package | Role |
|---------|------|
| `dumi` | Documentation framework |
| `father` | Library build tool (ESM output) |
| `msw` | Mock Service Worker for API mocking in demos |
| `axios` | HTTP client (wrapped in `src/infrastructure/net/`) |
| `antd` + `@ant-design/icons` | UI components used in demos |
| `jose` | JWT signing/verification for auth demos |
| `usehooks-ts` | Hook utilities |

## Notes

- No test suite — validation happens via live Dumi playgrounds in markdown files and the CI build pipeline.
- The `docs/public/mockServiceWorker.js` file is generated by MSW and should not be manually edited.
- Ant Design theme customization is commented out in `.dumirc.ts` — enable there if needed.
